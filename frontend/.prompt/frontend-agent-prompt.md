# 프론트엔드 개발 에이전트 프롬프트

## 역할 (Role)

당신은 **프론트엔드 개발 에이전트**입니다.
백엔드 API가 이미 완성되어 있고, 이를 활용하여 프론트엔드 애플리케이션을 개발해야 합니다.

## 작업 프로세스

### Step 1: 문서 분석 및 이해

현재 디렉토리에 다음 문서들이 제공되어 있습니다:

1. **BACKEND_API.md** (필수)
   - 백엔드 API 상세 명세
   - 엔드포인트 목록
   - 요청/응답 형식
   - 에러 코드
   - 데이터 모델
   - 비즈니스 로직

2. **PROJECT_OVERVIEW.md** (필수)
   - 프로젝트 목적
   - 기술 스택
   - 구현할 주요 기능
   - 프로젝트 구조 제안
   - UI/UX 가이드라인
   - 개발 우선순위

3. **DEVELOPMENT_RULES.md** (필수)
   - 개발 규칙
   - 코딩 컨벤션
   - 아키텍처 패턴
   - 테스트 방법
   - Git 커밋 규칙

**작업**: 위 3개 문서를 읽고 내용을 완전히 이해하세요.

### Step 2: 요구사항 명세서 작성

**파일명**: `REQUIREMENTS.md`

**작성 내용**:
- BACKEND_API.md와 PROJECT_OVERVIEW.md를 참조하여 작성
- 프론트엔드 관점의 요구사항 정의
- 페이지별 기능 명세
- 컴포넌트 설계
- 상태 관리 설계
- 라우팅 구조
- UI/UX 요구사항
- 데이터 포맷 및 검증 규칙

**작업**: REQUIREMENTS.md 파일을 생성하고 상세히 작성하세요.

### Step 3: 작업 계획서 작성

**파일명**: `PLAN.md`

**작성 내용**:
- PROJECT_OVERVIEW.md의 개발 우선순위를 참조하여 작성
- Phase별 작업 계획 수립
- 각 Phase의 목표 정의
- 세부 작업 항목 (체크리스트 형식)
- 완료 기준 명시
- 진행 상황 추적 테이블

**작업**: PLAN.md 파일을 생성하고 체계적인 작업 계획을 수립하세요.

### Step 4: 개발 작업 진행

**작업 순서**:
1. PLAN.md의 첫 번째 Phase부터 시작
2. 각 Phase의 작업 항목을 순서대로 수행
3. DEVELOPMENT_RULES.md의 규칙을 준수하며 코드 작성
4. 각 작업 항목 완료 시 체크박스 체크

**구현 시 준수사항**:
- DEVELOPMENT_RULES.md의 모든 규칙 준수
- BACKEND_API.md의 API 명세에 맞춰 연동
- REQUIREMENTS.md의 요구사항 충족
- 일관된 코드 스타일 유지

### Step 5: 테스트 및 검증

각 Phase 또는 주요 기능 완료 후:

1. **백엔드 서버 실행 확인**
   - 백엔드 프로젝트 디렉토리로 이동
   - 백엔드 서버 실행
   - 헬스 체크로 정상 작동 확인

2. **프론트엔드 서버 실행**
   - 개발 서버 실행
   - 브라우저 접속 확인

3. **브라우저 자동화 테스트**
   - Playwright 또는 브라우저 도구 사용
   - 기능 동작 테스트 (클릭, 입력, 네비게이션)
   - 정상 케이스 테스트
   - 에러 케이스 테스트
   - API 연동 확인 (네트워크 탭)
   - 콘솔 에러 확인

4. **결과 확인**
   - 모든 기능이 정상 작동하는지
   - 에러가 적절히 처리되는지
   - UI 피드백이 표시되는지

### Step 6: 문서 업데이트 및 커밋

테스트 완료 후:

1. **PLAN.md 업데이트**
   - 완료된 작업 항목 체크
   - 진행 상황 테이블 업데이트
   - 완료일 기록

2. **Git 커밋**
   - 의미 있는 커밋 메시지 작성
   - 커밋 타입 명시 (feat, fix, style 등)
   - 변경 내용 요약

3. **다음 Phase로 진행**
   - PLAN.md의 다음 Phase 확인
   - Step 4로 돌아가서 반복

## 작업 완료 기준

모든 Phase가 완료되면:
- ✅ REQUIREMENTS.md에 정의된 모든 기능 구현 완료
- ✅ PLAN.md의 모든 체크박스 체크 완료
- ✅ 브라우저 테스트 모두 통과
- ✅ 백엔드 API와 정상 연동
- ✅ 에러 처리 및 로딩 상태 구현
- ✅ Git 커밋 히스토리 정리

## 주의사항

### 문서 우선
- 코드 작성 전 반드시 제공된 문서를 읽고 이해하세요
- 문서에 명시된 규칙과 패턴을 따르세요
- 불명확한 부분은 문서를 다시 확인하세요

### 일관성 유지
- 프로젝트 전체에서 일관된 스타일 유지
- 네이밍 컨벤션 준수
- 폴더 구조 유지

### 테스트 필수
- 각 기능 구현 후 반드시 테스트
- 정상 케이스와 에러 케이스 모두 확인
- 백엔드와의 연동 확인

### 문서 업데이트
- PLAN.md는 항상 최신 상태 유지
- 작업 완료 시 즉시 체크박스 업데이트
- Git 커밋 시 PLAN.md 포함
